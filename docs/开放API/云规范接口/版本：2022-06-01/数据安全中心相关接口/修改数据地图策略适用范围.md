## 1. 接口描述

修改数据地图策略适用范围，私有化调用path为：/capi/DLPOpenApi/ModifyBackendPolicyScope，从8.6P1版本开始支持

## 2. 输入参数
| 参数名称 | 必选 | 类型 | 描述 |
|---------|---------|---------|---------|
| PolicyId | 否 | Integer | 策略列表唯一ID;;Required:true(只支持32位)<br/>示例值：1 |
| IncludeScope.N | 否 | Array of [ModifyBackendPolicyScopeData](/开放API/云规范接口/版本：2022-06-01/数据结构.md#ModifyBackendPolicyScopeData) | 应用的适用范围列表 |
| ExcludeScope.N | 否 | Array of [ModifyBackendPolicyScopeData](/开放API/云规范接口/版本：2022-06-01/数据结构.md#ModifyBackendPolicyScopeData) | 排除的适用范围列表 |

## 3. 输出参数

| 参数名称 | 类型 | 描述 |
|---------|---------|---------|
| Data | [ModifyBackendPolicyScopeRspData](/开放API/云规范接口/版本：2022-06-01/数据结构.md#ModifyBackendPolicyScopeRspData) | 业务响应数据<br/>注意：此字段可能返回 null，表示取不到有效值。|
| RequestId | String | 唯一请求 ID，由服务端生成，每次请求都会返回（若请求因其他原因未能抵达服务端，则该次请求不会获得 RequestId）。定位问题时需要提供该次请求的 RequestId。|

## 4. 示例

### 示例1 修改指定数据地图策略适用范围

#### 输入示例

```
POST / HTTP/1.1
Host: ioa.tencentcloudapi.com
Content-Type: application/json
X-TC-Action: ModifyBackendPolicyScope
<公共请求参数>

{
  "OsType": 0,
  "PolicyId": 49,
  "IncludeScope": [
    {
      "AssociatedObject": 1035,
      "Guid": "3b2606b7-3187-4089-8ad3-0b88c65e9062",
      "ID": 91,
      "ObjType": 0,
      "Operation": 1,
      "PolicyId": 49
    }
  ],
  "ExcludeScope": []
}
```

#### 输出示例

```json
{
    "Response": {
        "Data": {
            "PolicyId": 49
        },
        "RequestId": "web-76941297637216-17659489232"
    }
}
```
